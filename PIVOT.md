# Pivot Worksheet

## 1. Domain

ชื่อธุรกิจ: Workhair

ความเชื่อมโยงกับนักศึกษา: ธุรกิจของครอบครัว เป็นร้านทำผมที่มีลูกค้าถามข้อมูลซ้ำ ๆ ผ่านช่องทางออนไลน์ เช่น ราคา เวลาเปิด-ปิด การจองคิว และบริการที่เหมาะกับสภาพผม

## 2. ลูกค้า + ปัญหา

ลูกค้าหลักคือใคร: ลูกค้าที่ต้องการตัดผม ทำสี ดัด ยืด ทรีตเมนต์ หรือปรึกษาทรงผมก่อนเข้ารับบริการ

3 ปัญหาที่ระบบจะแก้:

1. ลูกค้าต้องรอแอดมินตอบคำถามพื้นฐาน เช่น ราคา เวลาเปิด-ปิด ที่ตั้ง และบริการที่มี
2. แอดมินต้องตอบคำถามซ้ำ ๆ หลายรอบ ทำให้เสียเวลาและอาจตอบไม่ทันในช่วงที่ร้านยุ่ง
3. ลูกค้าไม่แน่ใจว่าควรเลือกบริการหรือช่างคนไหน เช่น ทำสี ดัด ยืด ตัดทรงผู้ชาย หรืองานดูแลหนังศีรษะ

## 3. Differentiation

สิ่งที่ระบบนี้ทำได้ที่ MilkLab° template ทำไม่ได้:

1. ตอบคำถามเฉพาะร้านทำผมจากฐานความรู้ของ Workhair เช่น ราคาแพ็กเกจ ระยะเวลาทำผม นโยบายเลื่อนคิว และข้อมูลช่างแต่ละคน
2. ใช้ text embedding เพื่อค้นหาข้อมูลบริการที่ใกล้เคียงกับคำถามลูกค้า แม้ลูกค้าถามด้วยภาษาธรรมชาติ เช่น "ผมเสียทำอะไรดี" หรือ "อยากทำสีแต่ไม่อยากฟอก"
3. ช่วยแนะนำบริการหรือช่างให้เหมาะกับความต้องการเบื้องต้น เช่น งานเฟด งานทำสี งานตัดผมหญิง หรือทรีตเมนต์หนังศีรษะ
4. มีระบบ retry/backoff เพื่อป้องกันปัญหา quota หรือ rate limit ของ Gemini API ทำให้แอปเสถียรกว่า template พื้นฐาน
5. สามารถรันผ่าน Docker ได้ ทำให้ย้ายไป deploy หรือรันบนเครื่องอื่นได้ง่ายขึ้น
